# Free Replay 模拟交易

## Goal

在 Free Replay 中加入一个临时模拟交易模式，让用户在隐藏未来 K 线的复盘过程中练习开仓、挂单、平仓，并随着 K 线推进看到收益金额、收益率、胜率和盈亏比。

## Background

- Free Replay 现有入口在 `src/ui/FreeReplayPanel.tsx`，目前支持选择标的、开始时间、开始回放、前进和后退 K 线。
- Free Replay 的回放状态是页面内临时状态，`CONTEXT.md` 明确 Free Replay session 不持久化。
- `src/ui/App.tsx` 通过 `visibleCandlesForFreeReplay` 和 `freeReplay.cursorTime` 隐藏未来 K 线。
- Free Replay 图表显示 K 线和 instrument-level drawings，不显示历史 Trade entry/exit markers。
- 这个需求应沿用现有 Free Replay 加载 K 线和隐藏未来 K 线的逻辑，只在回放过程中增加模拟交易状态和右侧操作面板。

## Confirmed Decisions

- MVP 只支持单仓位：同一时间最多一个持仓。
- MVP 不支持加仓、补仓、均价调整或同时多笔持仓。
- 第一版支持 Long/Short 单仓位：开仓时选择方向，同一时间只能持有一个方向的一笔仓位。
- MVP 不支持反手、多空同时持有或多仓位组合。
- Free Replay 页面右侧增加模拟交易面板。
- 模拟交易需要有“开始”按钮；点击后才进入模拟交易模式，开仓/平仓控件才可用。
- 点击“开始”不会重置 Free Replay cursor，而是从当前 cursor 开始模拟。
- 已经揭示过的 K 线只作为历史上下文，不能回溯触发新挂单。
- 限价单只由后续 reveal 出来的 K 线触发。
- 初始账户本金固定为 `1000 USDT`。
- 支持仓位比例和杠杆。
- 保证金 = `1000 USDT * 仓位比例`。
- 名义仓位 = `保证金 * 杠杆`。
- 成交数量 = `名义仓位 / 成交价`。
- 仓位比例默认 `100%`，提供 `10% / 25% / 50% / 100%` 快捷选项，并允许手动输入 `1% - 100%`。
- 杠杆默认 `1x`，提供 `1x / 2x / 3x / 5x / 10x / 20x` 快捷选项，并允许手动输入 `1x - 125x`。
- 开仓成交时固化该笔持仓使用的仓位比例和杠杆；之后修改控件只影响下一笔开仓。
- 单笔收益率 = 该笔 PnL / 该笔实际占用保证金。
- 总收益率 = 已平仓总收益金额 / 初始账户本金 `1000 USDT`。
- 市价开仓和市价平仓按当前 cursor K 线的 `close` 成交。
- 限价单只在后续 reveal 出来的 K 线满足 `low <= limit <= high` 时触发，并按限价成交。
- `ArrowLeft` rewind 只回退图表视图，不回滚模拟交易状态、挂单、成交、持仓或统计。
- 模拟交易面板提供“重置模拟”按钮，用于清空当前模拟交易 session 并重新开始。
- 待触发限价单允许手动取消。
- 同类限价单只保留一笔：无持仓时最多一个待开仓限价单，有持仓时最多一个待平仓限价单；提交新的同类限价单会替换旧挂单。
- 右侧面板展示当前 session 的已完成模拟交易简洁列表。
- 图表显示已成交的模拟开仓和平仓标记，格式复用 Trade Review 的 marker 样式：开仓 `开 price`、平仓 `平 price`，并按当前 timeframe 对齐到 K 线。
- 待触发挂单第一版只在右侧面板显示，不画到图表上。
- 当前持仓显示浮动盈亏，按当前 cursor K 线 `close` 估算；浮动盈亏不计入已实现收益金额、总收益率、胜率或盈亏比。
- 重新开始 Free Replay、切换标的或切换开始时间时，自动重置模拟交易 session，清空挂单、持仓、已完成交易和统计。
- Free Replay 模拟交易记录第一版只保存在当前页面 session，不写入 SQLite，也不生成正式 Trade。

## Requirements

### R1. 模拟交易入口

Free Replay 中提供右侧模拟交易面板。未点击“开始”前，面板展示初始参数和开始入口；开始后进入模拟交易模式，显示可用操作、当前挂单、当前持仓和统计数据。

面板需要提供重置模拟入口。重置后清空挂单、持仓、已完成模拟交易和统计，但不改变当前 Free Replay 标的、时间或 cursor。

当用户重新开始 Free Replay、切换标的或切换开始时间时，系统自动重置模拟交易 session，避免旧标的或旧时间段的持仓、挂单和统计污染新的回放上下文。

### R2. 开仓

模拟交易模式开始后，用户可以选择 Long 或 Short，并提交市价开仓或限价开仓。

- 市价开仓立即按当前 cursor K 线 `close` 成交。
- 限价开仓成为待触发挂单。
- 待触发挂单只检查开始模拟之后新揭示的 K 线；当 `low <= limit <= high` 时按限价成交。
- 单仓位限制下，已有持仓时不能再开仓。
- 单仓位限制下，不支持反手开仓；需要先平掉当前持仓。
- MVP 不支持同时多个待开仓单；若已有待开仓单，应先取消或替换。
- 待开仓限价单可以取消；提交新的待开仓限价单会替换旧单。

### R3. 平仓

有持仓后，用户可以提交市价平仓或限价平仓。

- 市价平仓立即按当前 cursor K 线 `close` 成交。
- 限价平仓成为待触发挂单。
- 待触发平仓单只检查后续新揭示的 K 线；当 `low <= limit <= high` 时按限价成交。
- 待平仓限价单可以取消；提交新的待平仓限价单会替换旧单。
- 平仓后生成一笔已完成模拟交易，并更新统计数据。

### R4. 仓位比例与杠杆

模拟交易面板提供仓位比例和杠杆设置。

- 仓位比例基于固定账户本金 `1000 USDT`。
- 仓位比例默认 `100%`，提供 `10% / 25% / 50% / 100%` 快捷选项，并允许手动输入 `1% - 100%`。
- 杠杆默认 `1x`，提供 `1x / 2x / 3x / 5x / 10x / 20x` 快捷选项，并允许手动输入 `1x - 125x`。
- 需要展示或内部计算保证金、名义仓位和成交数量。
- 开仓成交时固化该笔持仓使用的仓位比例和杠杆；参数变更只影响后续开仓，不回改已经成交的持仓或历史统计。

### R5. 统计

模拟交易面板展示当前 session 的统计。

- 已实现收益金额：所有已平仓模拟交易的 PnL 合计。
- 单笔收益率：该笔 PnL / 该笔实际占用保证金。
- 总收益率：已平仓总收益金额 / 初始账户本金 `1000 USDT`。
- 胜率：盈利交易数 / 已平仓交易数。
- 盈亏比：平均盈利金额 / 平均亏损金额绝对值；没有足够样本时显示 `-`。
- 当前持仓浮动盈亏按当前 cursor K 线 `close` 估算。
- 浮动盈亏只展示在当前持仓信息中，不计入已实现收益金额、总收益率、胜率或盈亏比。

右侧面板展示当前 session 的已完成交易列表，每条至少包含方向、开仓价、平仓价、PnL、单笔收益率。

PnL 公式：

- Long：`(exitPrice - entryPrice) * quantity`。
- Short：`(entryPrice - exitPrice) * quantity`。

### R6. 图表标记

Free Replay 图表显示当前模拟 session 中已经成交的开仓和平仓标记。

- 标记格式复用 Trade Review 的 `tradeMarkers` 风格：开仓文本 `开 price`，平仓文本 `平 price`。
- Long 开仓在 K 线下方使用向上箭头，Long 平仓在 K 线上方使用向下箭头。
- Short 开仓在 K 线上方使用向下箭头，Short 平仓在 K 线下方使用向上箭头。
- 标记颜色与 Trade Review 保持一致：开仓 `#FACC15`，平仓 `#38BDF8`。
- 标记时间按当前 timeframe 对齐到 K 线，沿用现有 `markerTimeForEvent` 思路。
- 待触发挂单不画到图表上，只在右侧面板显示。

## Acceptance Criteria

- [ ] Free Replay 右侧出现模拟交易面板，且不破坏现有标的选择、开始时间、reveal/rewind 逻辑。
- [ ] 点击“开始”后，从当前 cursor 开始记录模拟交易事件，已揭示 K 线不会回溯触发订单。
- [ ] 用户可以设置仓位比例和杠杆，并按 `1000 USDT` 本金计算保证金、名义仓位和成交数量。
- [ ] 仓位比例支持快捷选项和 `1% - 100%` 手动输入，杠杆支持快捷选项和 `1x - 125x` 手动输入。
- [ ] 开仓后，该笔持仓的仓位比例和杠杆被固化，后续控件修改不影响已成交持仓。
- [ ] 用户可以市价开仓、限价开仓、市价平仓、限价平仓。
- [ ] 开仓支持 Long/Short 方向，但同一时间只能有一个方向的一笔持仓。
- [ ] 限价单只在后续 reveal 的 K 线满足 `low <= limit <= high` 后按限价成交。
- [ ] 待触发限价单可取消，同类新限价单会替换旧限价单。
- [ ] 单仓位约束生效：已有持仓时不能再次开仓，MVP 不支持加仓。
- [ ] `ArrowLeft` rewind 只回退图表视图，不回滚模拟交易状态。
- [ ] 用户可以重置模拟交易 session，重置不改变 Free Replay 标的、时间或 cursor。
- [ ] 重新开始 Free Replay、切换标的或切换开始时间时，模拟交易 session 自动重置。
- [ ] 平仓后统计收益金额、单笔收益率、总收益率、胜率和盈亏比。
- [ ] 当前持仓显示按 cursor K 线 `close` 估算的浮动盈亏，且不计入已实现统计。
- [ ] 右侧面板展示当前 session 的已完成交易简洁列表，至少包含方向、开仓价、平仓价、PnL、单笔收益率。
- [ ] 图表显示已成交模拟开仓和平仓标记，样式和 Trade Review marker 保持一致。
- [ ] 待触发挂单只在右侧面板显示，不画到图表上。
- [ ] Free Replay session 刷新页面后不保留模拟交易状态。
- [ ] 相关逻辑有聚焦测试覆盖，包括限价触发、单仓位约束和统计公式。

## Out Of Scope

- 多仓位、多笔并行订单、加仓、减仓、反手。
- 多空同时持有。
- 手续费、资金费、滑点、强平价和保证金风险模型。
- 把模拟交易保存到 SQLite 或导出为正式 Trade。
- 修改 Trade Review 或已有历史 Trade 统计。
- 回放回退时完整回滚模拟交易历史事件；MVP 采用 append-only 交易状态。
